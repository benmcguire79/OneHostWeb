Initial Firewall Setup

‍

OneHost loud's VPS’s come with firewall setup enabled by default. All incoming are blocked by default and all outgoing allowed. So you will be unable to access your VPS for any incoming such as SSH or RDP till you have created some firewall rules to allow access in the client area dashboard. This is a brief setup guide to help you understand how to create rules using SSH and RDP for the examples, other services wish to access you apply the same process just setting ports and protocol as required.

‍

Introduction

First login into your client area of Aeroneave, this will load the dashboard, listing your services. Click on the VPS which to use.

This will load into the VPS configuration area of your selected VPS. It will show status and other options which are covered in other knowledge base documents. For now just click the firewall on side panel.

This will take you to the firewall configuration page of the VPS. As can see a fresh setup VPS has no rules created. So click the add new rule button.

This will open the new rule creation window in your browser. You can view the protocol types for setting rules by selecting the drop down menu for it.

SSH

Since we wish to open SSH access we select TCP. Set the Destination port to 22, the default SSH port. Hit submit and the rule will be create.

As can see the rule is created, however this rule is not the most secure as will allow SSH from any IP.

Locking Down SSH Allowed

We can lock down access to SSH or any service if you know the IP will be connecting from. Let’s edit the rule so only one IP is allowed access, in the example used 123.123.123.123 SHOULD NOT BE USED but replaced with your desired public IP.

Hit submit and now we have a much more secure access to VPS only allowing the set IP access and all others will be blocked. If have more than one IP wish to use, simply create another rule/s same as have done just supplying different source IP.

RDP

For RDP we repeat the process simply changing the port number to 3389 as shown, set protocol to TCP and hit submit.

Now will have two rules created allowing access via SSH and RDP from any IP. See above on locking down the source IP allowed.

For another services follow the same steps simply changing to the listening port number of service and protocol type.
